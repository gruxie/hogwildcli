# Narrative Framework Reference

## Core Principles

### 1. Fidelity First
- Do not invent evidence
- Do not exaggerate prevalence or certainty
- Do not suppress ambiguity or minority views

### 2. Narrative Discipline

All outputs follow: **Context → Tension → Insight → Implication → Resolution**

### 3. Responsible Storytelling

Use storytelling to clarify, humanize, and make patterns memorable.

Do NOT:
- Let anecdotes override patterns
- Imply representativeness without support
- Use emotion as evidence

### 4. Source-of-Truth Rules

Distinguish:
- **Evidence** → what was observed
- **Insight** → interpretation
- **Recommendation** → action
- **Speculation** → forward-looking inference

Preserve: uncertainty, contradictions, minority views, limitations.

## Recommendation Structure

Each recommendation must include:
- Problem
- Why it matters
- Recommendation
- Expected effect
- Confidence / caution

## Integrity Review Format

### Status
- **Pass** — no transformation risks detected
- **Pass with cautions** — minor issues corrected automatically
- **Needs review** — issues require user judgment

### Issues Found (per issue)
- **Type** — prevalence inflation / confidence laundering / anecdote elevation / hypothesis suppression / causality overreach / minority view flattening
- **Claim** — the exact sentence
- **Source conflict** — the data that contradicts it
- **Action** — corrected automatically / requires user decision

### Changes Made
- Bulleted list of corrections with before/after wording

### User Decisions Needed
- Questions requiring researcher judgment before artifact is shareable
